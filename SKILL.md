# Web Verify

**铁律：涉及就业市场、技术趋势、薪资、学习路径、技术选型时，先 WebSearch 再回答。不依赖训练数据（2025年8月截止）。搜不到坦白告知。**

## 三步工作流

### 1. 搜 — 按意图选查询

按用户意图选择 BOSS直聘 JD、GitHub releases、公司财报、中国信通院等一手源。搜不到不硬搜——告诉用户"建议直接搜 [平台] [关键词]"。

**对抗验证**：对关键发现追加反向搜索 `{发现} 不准确 OR 替代 OR 争议`。发现矛盾标注 [DISPUTED]。

### 2. 滤 — 只认可靠源

- 公司招聘页/财报/GitHub → 可用
- CSDN/51CTO/掘金 职业/薪资/市场文 → 不可用（纯技术教程除外）
- 头条号/百家号/自媒体 → 不可用
- 标题含"逆袭""翻倍""年薪百万""收藏""必看" → 不可用
- "据统计"不注明机构名+报告名+时间 → 不可用
- 无源精确数字 → 不可用

**全垃圾源降级模板**：⚠️ 未找到可靠信息 → 建议自行验证 [平台+搜索词]

### 3. 呈 — 输出结构

1. 可信发现（仅可靠源，标注 [PRIMARY]/[CREDIBLE]/[UNVERIFIED]）
2. 事实 vs 推断（明确标注推理部分）
3. 你可做的事（具体搜索词/仓库/财报章节）
4. Sources

## 底线

- 市场/趋势类不单靠训练数据
- 从系统日期取年份，不硬编码
- 范围：IT 和 AI
- 禁止过度自信措辞（"显然""毫无疑问""绝对"）
